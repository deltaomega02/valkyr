# VALKYR — 암호화폐 자동매매 시스템 1세대 (아카이브)

> 자동매매 시스템 7세대 진화의 시작점: **VALKYR** → ARGOS → OMNI → METIS → HERMES → KAIROS → ATHENA
> 전체 계보: [github.com/deltaomega02](https://github.com/deltaomega02)

첫 자동매매 시스템. Ver 3.0부터 Ver 10까지 반복 개선하며 자동매매의 기본 구조
(시그널 판단 → 주문 실행 → 포지션 모니터링)를 처음 만들어 본 세대다.
이 저장소는 각 버전을 폴더째 보존한 아카이브 — 코드 품질보다 **시행착오의 기록**으로 의미가 있다.

## 기술 스택

Python · Upbit API (현물) · Streamlit (대시보드) · OpenAI API (후기 버전)

## 버전별 진화

| 버전 폴더 | 시도한 것 |
|---|---|
| `Ver_3.0.0` ~ `Ver_3.5.1`, `Ver_3.6_Adaptive` | 기술지표 기반 매수/매도 룰, 파라미터 손튜닝 반복 |
| `Ver_4_Adaptive` | 시장 상황에 따라 파라미터를 바꾸는 적응형 시도 |
| `Ver_5_Safety` | 손절·안전장치 도입 — "수익보다 생존" 관점의 시작 |
| `Ver_6` ~ `Ver_7` | 적응형 개선 + 단기 트레이딩 변형 |
| `Ver_8_Recalibrate` | 전면 재보정 + Streamlit 모니터링 대시보드 |
| `Ver_9_Origin` ~ `Ver_10` | 구조 재정리. 다단계 목표가 컨셉 — 이후 7세대 ATHENA에서 실현됨 |

## 이 세대가 다음 세대에 넘긴 것

- 단일 파일 스크립트의 한계 → 이후 세대의 모듈 분리 구조
- 손튜닝의 한계 → 5세대(HERMES)의 백테스트 전수 검증으로 발전
- 다단계 목표가 아이디어 → 7세대(ATHENA)에서 실현
